यह पोस्ट `Academic-Portfolio` ब्लॉग के लिए एक लाइव markdown संदर्भ है। यदि आप मुख्य रूप से यह देखना चाहते हैं कि कोड ब्लॉक कैसे दिखते हैं, तो सीधे नीचे "कोड ब्लॉक" अनुभाग पर जाएँ।

## शीर्षक और इनलाइन फ़ॉर्मेटिंग

Markdown **बोल्ड पाठ**, *इटैलिक पाठ*, ***संयुक्त ज़ोर***, ~~स्ट्राइकथ्रू~~, और इनलाइन कोड जैसे `npm run build` को संभालता है।

मानक इनलाइन लिंक काम करते हैं, जैसे [Astro](https://astro.build/), और संदर्भ-शैली लिंक भी लंबे लेखन में अच्छे से काम करते हैं, जैसे [Astro प्रलेखन][astro-docs]। https://docs.astro.build जैसे खुले URL भी इस सेटअप में लिंक के रूप में रेंडर होते हैं।

## उद्धरण ब्लॉक

> Markdown तब सबसे अच्छा काम करता है जब स्रोत पठनीय रहे और रेंडर किया गया पृष्ठ देखने में सुखद हो।
>
> यह इसे तकनीकी नोट्स, शैक्षणिक लेखन, चेंजलॉग और ट्यूटोरियल के लिए विशेष रूप से उपयोगी बनाता है।
>
> <cite>Academic Portfolio markdown प्रदर्शन</cite>

## सूचियाँ

### अक्रमित और क्रमित सूचियाँ

- `src/content/blog` में पोस्ट सामग्री का मसौदा तैयार करें।
- शीर्षक जोड़ें ताकि विषय-सूची में उपयोगी एंकर हों।
- ब्राउज़र में रेंडर किए गए आउटपुट की समीक्षा करें।

1. लेख का frontmatter लिखें।
2. नमूना markdown ब्लॉक जोड़ें।
3. साइट बिल्ड करें और पुष्टि करें कि पृष्ठ सही ढंग से रेंडर हो रहा है।

### नेस्टेड सूचियाँ

- Markdown नेस्टेड संरचना को भी संभाल सकता है।
  - यह रूपरेखाओं के लिए उपयोगी है।
  - यह उप-बिंदुओं के साथ चरणों को दस्तावेज़ित करने में भी सहायक है।
- यहाँ उद्देश्य केवल रेंडरिंग को जाँचने में आसान बनाना है।

## कोड ब्लॉक

निम्नलिखित उदाहरण वर्तमान थीम में विभिन्न फ़ेंस्ड कोड ब्लॉक शैलियों की तुलना करना आसान बनाते हैं। प्रत्येक ब्लॉक अब बाईं ओर एक भाषा पिल और ऊपरी-दाएँ किनारे पर एक कॉपी बटन दिखाता है।

```bash
npm install
npm run dev
npm run build
```

```ts
type Publication = {
  title: string;
  year: number;
  tags: string[];
};

const recentPublications = (items: Publication[]) =>
  items.filter((item) => item.year >= 2024);
```

## Markdown में HTML

जब आपको किसी ऐसी चीज़ की आवश्यकता हो जो अकेले markdown स्पष्ट रूप से व्यक्त नहीं कर सकता, तो कच्चा HTML भी उपलब्ध है।

<details>
  <summary>विस्तार योग्य टिप्पणी</summary>
  <p>यह प्रकटीकरण, FAQ अनुभागों, या किसी पोस्ट के अंदर वैकल्पिक कार्यान्वयन टिप्पणियों के लिए उपयोगी है।</p>
</details>

## फ़ुटनोट्स

फ़ुटनोट उद्धरणों, साइड टिप्पणियों, या त्वरित स्पष्टीकरणों के लिए सुविधाजनक हैं।[^1]
